## 1. 틸노트 (Tilnote) — 11장

| # | 슬라이드 제목 | 핵심 메시지 | 비주얼 추천 |
|---|---|---|---|
| 1 | 커버 | 가입자 10명 중 8명이 글을 쓰지 않는다 | 틸노트 에디터 빈 화면 스크린샷 + 83.5% 숫자 타이포 강조 |
| 2 | 내 역할 | 단독 총괄, GA4 기반 전 과정 주도 | 타임라인 (12월~3월) + 담당 영역 텍스트 |
| 3 | Problem — 퍼널 | 가입까지 왔는데 글을 쓰지 않는다 | 3단계 퍼널 다이어그램 (346,881 → 1,519 → 250) — 숫자 낙차가 핵심 |
| 4 | Problem — 추가 근거 | 에디터에 들어와도 30초 내 이탈 | 온보딩 스킵 54% / 완료율 12.7% / 세션 26~35초 — 수치 3개 카드형 나열 |
| 5 | Hypothesis | 3개 가설 한눈에 | 가설 3개 번호형 리스트 (Clip / 온보딩 / 공간 분리) |
| 6 | Design Judgment 1 — Clip | 빈 화면이 아예 안 나오게 | Before(빈 에디터 진입) → After(Clip 경유 진입) 플로우 비교 |
| 7 | Clip 작동 방식 | 유튜브/웹 → AI 요약 → 에디터 진입 10초 | tilnote.io/clip 실제 화면 + 3단계 플로우 화살표 (외부 콘텐츠 → Clip 요약 → 에디터) |
| 8 | Design Judgment 2 — 예시 프롬프트 | 첫 행동 유도형 선택지 + 온보딩 미해결 한계 명시 | Agent 패널 Before(빈 입력창) → After(예시 프롬프트 3개) 스크린샷 |
| 9 | Design Judgment 3 — 공간 분리 | 지시하는 곳 vs 쓰는 곳 | Before/After 레이아웃 비교 (웹 + 모바일 Drawer 각각) |
| 10 | Evidence | 설문·PO·GA4 근거 묶기 | 설문 응답률 bar chart 3개 (Q4 55.6% / Q7 33.3% / Q2 44.4%) + PO 인용 텍스트 박스 2개 |
| 11 | Reflection | 측정 없이 배포한 실수 + 배운 것 2개 | Before 데이터 공백 시각화 (GA 트래킹 시작일 타임라인) + "측정 설계 = 디자인 설계" 텍스트 강조 |

## 2. SK 휠비 × 로드스캐너 — 13장

| # | 슬라이드 제목 | 핵심 메시지 | 비주얼 추천 |
|---|---|---|---|
| 1 | 커버 | 두 앱의 진입 장벽을 허물어 보행 생태계 복구 | 휠비 + 로드스캐너 앱 아이콘 나란히 + 한 줄 요약 타이포 |
| 2 | 내 역할 | 단독 설계 → 외부 검증 위탁 | 역할 표 + 협업 구조 간단 다이어그램 (본인 → LBStech 위탁 화살표) |
| 3 | Problem — 생태계 교착 | 데이터 없음 → 휠비 못 씀 → 데이터 쌓을 이유 없음 | 악순환 루프 다이어그램 (휠비 ↔ 로드스캐너 원형 화살표) + 인용 텍스트 박스 |
| 4 | Problem A — 휠비 | 낮은 시인성 → 앱 학습 포기 → 길찾기 포기 | 낮은 대비의 파스텔 UI 스크린샷 + 문제 흐름 텍스트 (시인성 → 학습 포기 → 이탈) |
| 5 | Problem B — 로드스캐너 | UX 진입 장벽 → 없는 앱 취급 → 데이터 공백 | 로드스캐너 기존 화면 스크린샷 (복잡한 등록 플로우) + "없는 앱 취급" 텍스트 강조 |
| 6 | Hypothesis | 가설 2개 | 가설 카드 2개 나란히 (휠비 / 로드스캐너) |
| 7 | Design Judgment 1 — 컬러 | 브랜드보다 시인성 — 실무적 트레이드오프 | 파스텔 컬러 vs Safety-Pass Green / Warning-Obstacle Red Before/After + WCAG 4.5:1 명도 대비 수치 |
| 8 | Design Judgment 2 — Active Glyph | 수동적 상징 → 능동적 상징 | 기존 정지 장애인 마크 vs 15도 기울인 Active Glyph 나란히 + 설계 의도 한 줄 |
| 9 | Design Judgment 3 — UX 라이팅 | 언어가 행동을 만든다 | "장애물 저장하기" → "내가 처음으로 정보 수집하기" 텍스트 비교 + 다중 사진 업로드 Before/After 플로우 |
| 10 | Design Judgment 4 — Seamless Flow | 두 앱을 하나의 보행 경험으로 연결 | 로드스캐너 수집 → 휠비 30m 예고 알림 → 원터치 우회 경로 3단계 플로우 화면 |
| 11 | Impact | N=7 검증 결과 전원 완수 | 수치 4개 강조 카드 (교육완료 100% / 1회 설정 71% / 독립 완수 100% / 로드스캐너 100%) + LBStech 외부 위탁 명시 |
| 12 | Influence | 만장일치 채택 → SK 휠비 AR 공식 가이드라인 | "만장일치 의사결정" 텍스트 강조 + SK 휠비 AR 가이드라인 채택 결과 명시 |
| 13 | Retrospective | 로드스캐너 28% 한계 + Universal Design 교훈 | 휠비 71% vs 로드스캐너 28% 비교 bar + "제약을 원칙으로" 텍스트 |

## 3. 구루미닷컴 — 12장

| # | 슬라이드 제목 | 핵심 메시지 | 비주얼 추천 |
|---|---|---|---|
| 1 | 커버 | 2,000만원 가이드가 사장된 조직에서 브랜드 허브를 만들다 | 구루미닷컴 메인 화면 스크린샷 + 한 줄 요약 타이포 |
| 2 | 내 역할 | 이슈레이징부터 경영진 합의까지 | 역할 표 + 협업 구조 (디자이너·마케터 2인·FE 1인) |
| 3 | Problem — 외부 | 원하는 서비스를 못 찾고 이탈 | gooroomee.com = Camstudy 점유 사이트 구조 다이어그램 (방문자 유형 → 찾는 서비스 → 이탈) |
| 4 | Problem — 내부 | 2,000만원 가이드가 아무도 모른다 | 내부 고충 3개 카드 (마케터: URL 특정 불가 / 기획자: Google Drive 공유 / 외주 가이드: 2,000만원 사장) |
| 5 | Hypothesis | 가설 2개 | 가설 카드 2개 나란히 (조직 내재화 / 허브 구조) |
| 6 | Design Judgment 1 — 이슈레이징 | 혼자 시작한 것을 조직 어젠다로 | 3단계 타임라인 (개인 작업 시작 → 내부 논의 시점 포착 → 경영진 합의) |
| 7 | Design Judgment 2 — 최소 기준 + 루트 | 규칙이 아니라 대화로 내재화 | 방대한 외주 가이드(두꺼운 문서 이미지) vs 최소 기준 4개(컬러·서체·로고·Do/Don't) 대비 + 브랜드전략팀 루트 화살표 |
| 8 | Design Judgment 3 — 브랜드 언어 | 3개 키워드 → 시각 언어로 번역 | 3열 표: 든든(Glassmorphism 화면) / 가까운(Clay 오브젝트) / 넉넉한(영상 배경 스크린샷) |
| 9 | 비주얼 방향 | 브랜드 언어가 화면에 구현된 모습 | 구루미닷컴 실제 화면 — Glassmorphism·Clay·영상 배경 섹션별 스크린샷 나열 |
| 10 | Impact | 70% 만족 + CVR 23% + VoC 감소 | 수치 3개 강조 카드 (만족도 70% / CVR +23% / VoC 15~20→3건 이하) + 컨펌 루프 3→1 시각화 |
| 11 | Influence | 조직이 브랜드를 필요로 하게 만들었다 | 브랜드전략팀 중심 커뮤니케이션 루트 다이어그램 (기존: 각 팀 → 디자이너 / 변경: 각 팀 → 브랜드전략팀 → 디자이너) |
| 12 | Retrospective | 피드백 루프 부재 + 템플릿 한계 → One Source Multi-Use | 템플릿 배포 방식 한계 텍스트 + "텍스트만 바꿔 즉시 산출물" One Source Multi-Use 개념 다이어그램 |

## 공통 주의사항

- **Design Judgment 슬라이드에 실제 화면 스크린샷 필수.** Before/After가 있는 슬라이드는 텍스트 설명보다 실제 화면이 설득력의 핵심입니다.
- **수치 강조는 숫자 하나를 크게.** 83.5%, 100%, 23% 같은 핵심 수치는 슬라이드 중앙에 크게 배치하는 방식이 면접관 시선을 잡습니다.
- **다이어그램 우선 슬라이드**: 틸노트 퍼널(#3), SK휠비 악순환 루프(#3), 구루미 사이트 구조(#3) — 텍스트 설명보다 다이어그램 하나가 훨씬 강합니다.
